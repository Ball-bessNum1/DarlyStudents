# Darly Student
Дневник-расписание на базе 1С

[![1C:Enterprise](https://img.shields.io/badge/1C:Enterprise-8.3-yellow.svg)](https://1c.ru/)
[![Version: 1.0](https://img.shields.io/badge/Version-1.0-green.svg)]()

Конфигурация на платформе «1С:Предприятие 8» для автоматизации учета учебных занятий, визуализации расписания и контроля посещаемости. 

Изначально проект создавался для удобного ведения расписания университетской группы, поэтому в логику заложены специфические образовательные механики (чередование недель, фиксированное время пар), но архитектура легко масштабируется под любые задачи планирования. Конфигурация успешно протестирована как в среде Windows, так и на Arch Linux.

## 🚀 Основной функционал (Версия 1.0)

В первой версии (v1.0) реализован базовый, но полностью рабочий каркас системы:

* **Интерактивный планировщик:** Визуальное отображение занятий с помощью элемента «Планировщик». Реализован точный расчет длительности пар (с преобразованием времени относительно начала дня) для корректного отображения на шкале времени без дублирования и "наползания" суток.
* **Умное автозаполнение:** Разработан алгоритм автоматической генерации документов расписания. Код анализирует прошлые периоды и автоматически копирует расписание на 14 дней вперед с учетом чередования недель (числитель/знаменатель). Алгоритм автоматически пропускает воскресенья, экономя ресурсы системы.
* **Чистота интерфейса и данных:** Внедрен механизм жесткой фильтрации пустых окон или отмененных занятий (объектов `null`). Фильтрация происходит как на уровне вывода элементов в Планировщик, так и на уровне СКД-запросов, не позволяя "мусору" попадать в аналитику.
* **Аналитика посещаемости:** Учет посещений реализован на базе регистров накопления. Написаны оптимизированные запросы (с приведением типов `ТИПЗНАЧЕНИЯ`), которые корректно суммируют посещаемость по предметам, даже если в базе присутствуют смешанные типы данных (строки и ссылки).

## 🛠 Технологии и инструменты
* Платформа: **1С:Предприятие 8 (Учебная версия)**
* СУБД: Файловая база данных
* Язык запросов: SQL-подобный язык запросов 1С
* Инструменты отчетности: СКД (Система компоновки данных)

## ⚙️ Установка и запуск

Проект поставляется в виде исходных файлов конфигурации (XML) для удобного отслеживания изменений в Git, а также (опционально) в виде файла выгрузки информационной базы.

**Вариант 1 (Загрузка через исходники):**
1. Создайте пустую информационную базу в Конфигураторе 1С.
2. В верхнем меню выберите `Конфигурация` -> `Загрузить конфигурацию из файлов...`.
3. Укажите путь к корневой папке репозитория и дождитесь окончания загрузки.
4. Нажмите `F7` (Обновить конфигурацию базы данных).

**Вариант 2 (Через .dt файл, если вы добавите его в релизы):**
1. Создайте пустую информационную базу.
2. Откройте Конфигуратор, выберите `Администрирование` -> `Загрузить информационную базу`.
3. Выберите скачанный `.dt` файл.
